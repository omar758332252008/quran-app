# مصحف التلاوة الذكي

تطبيق React يعرض القرآن الكريم (114 سورة، 6236 آية بالرسم العثماني) ويستخدم التعرف الصوتي بالمتصفح (Web Speech API) عشان يسمع تلاوتك ويصحح لك كلمة كلمة.

## التقنيات

- **الفرونت إند**: React + TypeScript + Vite + Tailwind + shadcn/ui
- **قاعدة البيانات**: Supabase (Postgres) - بيانات قراءة فقط، بدون أي سيرفر باك إند، الفرونت إند يتصل بسوبابيز مباشرة عبر `@supabase/supabase-js`
- **التعرف الصوتي**: Web Speech API المدمجة في المتصفح (Chrome / Edge)

## التشغيل محليًا

```bash
npm install
npm run dev
```

المتغيرات المطلوبة موجودة في `.env` (مفتاح anon العام، آمن للاستخدام في الفرونت إند):

```
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...
```

## البناء للنشر

```bash
npm run build
```

الناتج بيظهر في `dist/` - ملفات استاتيك بحتة، تقدر تنشرها على أي استضافة (Vercel, Netlify, Cloudflare Pages, GitHub Pages...).

## قاعدة البيانات (Supabase)

جدولين فقط في schema `public`:

- **surahs**: بيانات السور (114 سجل) - الاسم، عدد الآيات، مكان النزول...
- **ayahs**: نص الآيات (6236 سجل) بالرسم العثماني، مرتبطة بـ `surah_id`

كلاهما RLS مفعّل مع policy للقراءة العامة فقط (`select` لـ `anon`/`authenticated`)، البيانات مرجعية وما فيها أي كتابة من المستخدمين.

## ملاحظات على آلية التصحيح الصوتي

- المقارنة بين الكلمات بتتم **بالترتيب** (كلمة بكلمة في نفس الموضع)، مش بحث "هل الكلمة دي موجودة في أي مكان من كلامك" - ده اللي يخلي التصحيح يطلعلك الكلمة الغلط في مكانها الصحيح.
- لو المايك سكت تلقائيًا بسبب صمت قصير، التطبيق بيعيد تشغيله لوحده طول ما لسه في وضع "استماع".
- لما تضغط إيقاف، التطبيق بياخد وقته يستقبل آخر كلمة وصلت من المايك قبل ما يحسب النتيجة، عشان النتيجة تكون دقيقة.
